Technical setup (operators)

Waitlist form (recommended — no Google)
1. On Render (scanner): SYNTRIX_WAITLIST_INGEST_SECRET — long random string (e.g. openssl rand -hex 32).
2. On Netlify: WAITLIST_SCANNER_INGEST_URL = your API + /api/public/waitlist  e.g. https://api.syntrix.solutions/api/public/waitlist
3. On Netlify: WAITLIST_INGEST_SECRET = exact same string as step 1 (sent as Bearer from the Netlify function). Redeploy Netlify.
Fallbacks: webhook / Resend / Apps Script — see netlify/functions/signup-notify.mjs.

Live scans require sign-in JWT (separate from the waitlist row): Netlify build AUTH0_DOMAIN, AUTH0_CLIENT_ID, AUTH0_AUDIENCE; Scanner AUTH0_DOMAIN, AUTH0_AUDIENCE. Callback https://syntrix.solutions/auth-callback.html. Until Stripe works: SYNTRIX_BILLING_REQUIRED=false. Temporary last resort only: SYNTRIX_AUTH_REQUIRED=false exposes scans without login — do not leave on public production.

Optional SYNTRIX_API_BASE in Netlify build; or assets/js/auth-overlay.js after auth-config.js.
