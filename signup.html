Technical setup (operators)

Security: Webhook URLs, deployment tokens, and API secrets belong only in Netlify / Render environment variables (server-side). Do not commit them to git or put them in HTML, JS, or CSS served to visitors — the browser never needs your Apps Script or ingest URL.

Waitlist delivery is configured in netlify/functions/signup-notify.mjs (scanner ingest, webhook, email, or legacy script). After changing env vars, trigger a production deploy.

Password sign-up requires SYNTRIX_PASSWORD_AUTH=true on the scanner and a matching Netlify build (npm run build writes auth-config.js). Set SYNTRIX_PASSWORD_AUTH=false on Netlify only if you intentionally disable password UI.
